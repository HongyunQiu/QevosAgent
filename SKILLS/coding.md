# SKILL: 代码开发

适用领域：软件工程、编程、代码审查、调试、重构。

## 代码质量规范

- 优先可读性：变量/函数名清晰表达意图，不写缩写
- 每个函数只做一件事，超过 50 行考虑拆分
- 错误必须显式处理，不允许裸 `except: pass`
- 写代码前先写测试用例或验证逻辑（TDD 思维）

## 调试流程

1. 复现问题：先用最小用例复现，不在复杂环境里盲调
2. 定位原因：用 `run_python` 或 `shell` 打印中间状态
3. 修复验证：修复后重新运行原始失败用例确认通过
4. 记录结论：将根因写入草稿本，避免反复踩坑

## 文件产物规范

- 所有生成的代码文件写入 `$RUN_DIR/artifacts/`
- 测试脚本命名为 `test_<功能名>.py`
- 不直接修改项目源文件，除非任务明确要求

## 工具偏好

- 计算/逻辑验证 → `run_python`
- 文件操作/系统命令 → `shell`
- 大文件阅读 → `file_outline` + `read_file_lines`
- 跨文件搜索 → `grep_files`
