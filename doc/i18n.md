# 国际化（i18n）支持

## 背景

QevosAgent 原本所有面向终端、看板和 LLM 的字符串均硬编码为中文。为支持英文环境下的正常使用（API 输出、日志可读性、海外部署），引入了完整的双语 i18n 方案。

---

## 语言检测

检测优先级（高→低）：

1. 环境变量 `QEVOS_LANG=zh` / `QEVOS_LANG=en`（显式指定，优先级最高）
2. 系统 locale（`locale.getlocale()`）
3. 默认：`zh`

**关键约束**：`run_goal.py` 作为独立入口，其 `.env` 文件由 `ensure_env_defaults()` 在 `main()` 顶部加载。因此 `from agent.i18n import t` 必须放在 `ensure_env_defaults()` 调用之后（函数内部），否则模块级 `LANG` 检测时 `QEVOS_LANG` 尚未从 `.env` 读入。

Dashboard 侧（`server.js`）同样在 `.env` 加载块之后才计算 `LANG`，随后通过 `serveStatic()` 将 `window.QEVOS_LANG` 注入每个 HTML 响应的 `<head>` 中，供前端 JS 读取。

---

## 核心模块：`agent/i18n.py`

统一字符串仓库，所有字符串均以 `key: str` 索引，分 `zh` / `en` 两个表。

```python
from agent.i18n import t

t("loop.done")                          # → "✨ 完成！" / "✨ Done!"
t("rg.run_dir_hint", run_dir=run_dir)  # → 含参数插值
```

### 字符串分类

| 前缀 | 用途 | 消费方 |
|------|------|--------|
| `loop.*` | 终端进度输出（ConsoleHooks） | `loop.py` |
| `interrupt.*` | 用户干预命令终端输出 | `user_interrupt.py` |
| `status.*` / `log.*` | `/status`、`/log` 命令输出 | `user_interrupt.py` |
| `marker.*` | LLM↔Python 协议标记（工具结果头、advisor 注入等） | `loop.py` 生产，`persistence.py` + `server.js` 消费 |
| `compress.*` | 压缩 LLM 的系统提示和桥接消息 | `compression.py` |
| `note.*` | 草稿本自动笔记迷你 LLM 提示 | `compression.py` |
| `advisor.*` | 高级指导员上下文与注入文本 | `advisor.py` |
| `sys.*` | Agent 系统提示各节 | `llm.py` |
| `err.*` | JSON 格式错误反馈 | `llm.py` |
| `parse.*` | 内联解析错误注入 | `llm.py` |
| `rg.*` | `run_goal.py` 目标前缀与终端提示 | `run_goal.py` |
| `exec.*` | 工具执行错误消息（LLM 可见） | `executor.py` |

## marker 协议的双侧处理

`marker.*` 字符串由 Python（`loop.py`）生产并写入 `short_term`，由 `persistence.py` 和 `server.js` 消费解析。

**问题**：旧日志中存在中文标记，新代码切换到英文后无法解析历史记录。

**解法**：消费侧采用 OR 逻辑同时匹配中英文：

```python
# persistence.py
_FAILURE_MARKERS = ("执行失败", "execution failed", "[TOOL ERROR]")
any(m in content for m in _FAILURE_MARKERS)
```

```js
// server.js
const toolMatch = text.match(
  /^\[(?:工具|Tool):\s*([^\]]+)\]\s*(执行成功|执行失败|executed successfully|execution failed)\s*\n?([\s\S]*)/
);
```

## 前端 i18n：`ui_i18n.js` + `serveStatic()` 注入

### 注入流程

```
server.js serveStatic()
  → 读取 HTML 文件
  → 在 </head> 前插入 <script>window.QEVOS_LANG="zh";</script>
  → 返回修改后的 HTML
```

### `ui_i18n.js`

```js
window.uiT = function(key, vars) { /* 查表 + 插值 */ };
window.UI_LANG = 'zh' | 'en';
```

- 支持普通字符串和数组（如前缀过滤列表 `goal.filter.prefixes`）
- 插值语法：`uiT('episodic.count', { n: 5 })` → `"5 条"` / `"5 record(s)"`

### index.html 覆盖的字符串

| 位置 | 字符串 |
|------|--------|
| nostop 按钮 title | "持续对话模式：完成任务后不退出…" |
| 目标过滤前缀列表 | `['#', '【', '本次运行', '提示：', ...]`（含英文前缀） |
| nostop idle banner | "任务完成 — 请输入下一个目标" |
| ask_user 事件标签 | "💬 Agent 向你提问" |
| Skill 默认内容模板 | "# SKILL: …\n\n适用领域：\n\n## 规范\n\n" |
| memory_macro.md 缺失提示 | "memory_macro.md 尚不存在。" |
| 宏观记忆保存 toast | "✓ 宏观工作记忆已保存" |
| 情景记忆条数 | "${n} 条" |
| 情景记忆空状态 | "暂无记录" |
| 情景记忆 JSON 错误 | "✗ 第 N 行 JSON 无效" |

### view.html 覆盖的字符串

tab-bar 等待文字、empty-state 段落、附加图片按钮 title、聊天输入框 placeholder、发送按钮文字、用户标签"你"、图片 alt、移除按钮 title、图表渲染失败文字、空表格文字、`[Web用户]:` 前缀、发送失败提示。

## 实现批次

| 批次 | 内容 |
|------|------|
| Batch 1 | 终端交互字符串（loop、interrupt、status、log） |
| Batch 2 | LLM facing 字符串（marker、compress、note、advisor、sys、err、parse）；server.js OR 正则；persistence.py OR 逻辑；server.js `.env` 加载顺序修复；启动 banner 显示语言 |
| Batch 3 | `run_goal.py`（rg.*）、`executor.py`（exec.*）、前端 UI（ui_i18n.js + index.html + view.html） |

**跳过（暂不处理）**：`agent/tools/standard.py`（工具描述）、`AGENTS.md`、`ADVISOR.md`（用户可编辑文件，双语维护方案待定）。
